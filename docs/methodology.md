# Competitive Research Methodology

## Objective

Produce evidence-led competitor research for internet and AI products with enough product detail to support real decision-making.

## Core method

1. Clarify the task progressively
2. Define the competitor set and substitutes
3. Retrieve external evidence first
4. Read user-provided materials early, but do not treat them as truth
5. Separate facts, observations, and judgments
6. Prefer product detail over abstract strategic commentary
7. Stop when evidence is sufficient, not when the search space is exhausted

## Retrieval rules

- External retrieval is the default path for competitor facts, reviews, pricing, launches, market movement, and user sentiment.
- Local materials should only provide:
  - task constraints
  - existing project context
  - user-provided assets
- Important claims should be cross-checked against multiple sources when possible.

## Product-detail checklist

For each major product, try to capture:
- homepage or core product page
- feature pages or docs
- pricing page
- key workflows and UX details
- user-reported strengths and pain points
- launch or changelog evidence
- founder/team/company context when relevant
- funding or investor context when relevant
- visual evidence or screenshot references when useful

## Output structure

Recommended sections:
- Executive Summary
- Comparison Table
- Detailed Analysis by Product
- Strategy Implications (only when justified)
- Information Gaps and Disputes
- Sources

## Citation standard

Every high-impact conclusion should include:
- source marker
- date or time context
- credibility label
- evidence excerpt or faithful paraphrase
- the most relevant product link when available
